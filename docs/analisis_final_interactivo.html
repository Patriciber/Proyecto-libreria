Hex Arquitectura

Lumière ha evolucionado de un prototipo simple a una aplicación basada en Arquitectura Hexagonal, con una separación clara entre el Dominio, la Aplicación y la Infraestructura. El sistema actual soporta autenticación robusta y un ecosistema de calidad profesional.

Flujo Arquitectónico

graph TD subgraph "Capas de Lumière" Entry[Routers: auth.py, books.py] Logic[Services: auth_service, book_service] Interface[Ports: Repositories Interfaces] Data[Adapters: MockRepositories, SecurityAdapter] end Entry --> Logic Logic --> Interface Data -- Implements --> Interface

La arquitectura asegura que si mañana decidimos cambiar MockRepository por un PostgresRepository, la lógica de negocio en la capa de Aplicación no sufrirá ningún cambio.

Núcleo de Seguridad

  • Hashing de Contraseñas: Uso de bcrypt para almacenar passwords de forma segura.
  • Tokens JWT: Emisión de tokens firmados bajo el estándar HS256 con expiración automática.
  • Inyección de Dependencias: Control de acceso basado en el usuario autenticado para el endpoint /premium.

Laboratorio de Calidad

Se han implementado y configurado las siguientes herramientas en el ciclo de desarrollo:

Hypothesis

Tests basados en propiedades que generan cientos de casos de prueba aleatorios para encontrar errores extremos en la búsqueda.

Pylint

Análisis de la calidad del código, asegurando estándares de legibilidad y buenas prácticas de Python.

Bandit

Auditoría automática de seguridad para prevenir vulnerabilidades comunes de codificación.

Orquestación

Uso de pre-commit para ejecutar todas las verificaciones automáticamente al guardar cambios.
